Distributed Drawing Board (分布式标绘画板)
 
项目简介
 
本项目是基于鸿蒙软总线开发的跨设备协同标绘应用，在软总线基础通信能力上完成大量定制化开发与功能适配，核心实现分布式环境下的实时画板协同、离线数据同步及操作撤销功能，作为跨设备通信应用的落地载体，充分契合跨设备通信场景的业务需求。
 
项目结构

核心功能
 
1. 分布式协同标绘：基于鸿蒙软总线实现多设备实时绘图操作同步，支撑跨设备协作标绘
2. 离线重连同步：离线状态下本地存储绘图操作，设备上线后自动完成跨设备数据同步
3. 操作撤销：支持绘图操作的单步回退，提升标绘操作的容错率与使用体验
 
技术实现基础
 
- 开发底座：鸿蒙操作系统（HarmonyOS）
- 核心通信：基于鸿蒙软总线做定制化开发，依托其跨设备发现、连接、高速传输能力，自研适配标绘场景的通信逻辑
- 开发语言：JS/TS（UI交互层）、C/C++（鸿蒙原生能力调用/软总线定制开发层）
 
开发与使用约束规范
 
设备与环境约束
 
1. 系统版本：运行设备需为HarmonyOS 2.0及以上版本
2. 网络要求：跨设备协同需保证设备处于同一局域网，或支持鸿蒙软总线直连能力
3. 开发环境：需使用鸿蒙DevEco Studio，完成HarmonyOS开发环境的基础配置
 
权限规范
 
1. 必须授予应用设备连接权限，保障跨设备发现与通信
2. 必须授予应用存储读写权限，支持离线数据存储与上线同步
3. 跨设备协同时，需保证各设备应用的权限配置一致
 
代码开发规范
 
1. 遵循鸿蒙官方应用开发编码规范，原生层与JSUI层通过标准接口进行数据交互
2. 分布式通信相关代码统一归置至Distributed/模块，禁止跨模块冗余开发
3. 离线同步数据的序列化/反序列化逻辑统一在NetworkManagement/模块实现，保证数据格式一致性
4. 所有绘图操作日志需按统一格式记录，为操作撤销、数据同步提供统一数据支撑
 
安装与使用步骤
 
1. 克隆仓库：git clone 项目仓库地址
2. 环境配置：打开鸿蒙DevEco Studio，完成HarmonyOS SDK的对应版本配置
3. 项目导入：将克隆的项目文件夹导入DevEco Studio，自动完成依赖加载
4. 设备部署：将项目编译后，部署至目标鸿蒙设备（支持多设备同时部署）
5. 协同使用：启动应用后，通过设备发现功能关联待协同设备，关联成功后即可实现多设备标绘协同、离线同步等功能
