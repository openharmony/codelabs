# 联系人小应用 — 亮点与实现

## 一句话介绍
一个轻量的联系人管理小应用，支持联系人创建、查看、编辑、删除（CRUD），并以简洁的页面导航与模块化服务层实现，适合作为 ArkTS/Ability 框架下的小型示例或练手项目。

## 亮点
- 性能轻量：页面与数据与业务层分离，便于单测与复用。
- 简洁导航：通过路由常量与页面组件实现清晰的跳转流程，用户体验流畅。
- 可扩展性强：数据模型 `Contact` 与服务接口清晰，方便接入远程接口或本地持久化。
- 易读代码：类型化的 TypeScript/ArkTS 文件，使维护与扩展成本低。

## 技术实现要点
- 数据模型：`Contact`（位于 model 文件夹）定义了联系人字段与类型。
- 服务层：`ContactService` 负责CRUD操作与数据封装，页面通过调用其方法完成业务流程。
- 页面层：各页面文件（`ContactListPage.ets`、`ContactCreatePage.ets`、`ContactEditPage.ets`、`ContactDetailPage.ets`）负责 UI 与用户交互，尽量保持无状态，所有数据由服务层提供。**页面层支持拼音模糊查找和按字母搜索排序**。
- 路由/常量：使用 `Routes.ts` 管理页面路由常量，减少硬编码跳转地址。


## 未来扩展方向
- 持久化：将 `ContactService` 的内存数据改为使用本地存储或远程 API。
- 导入/导出：支持 vCard 或 CSV 的导入与导出功能。
- 多字段搜索与标签分组：增强检索能力与分类功能。
- 单元/集成测试：为 `ContactService` 与关键页面添加测试用例。
