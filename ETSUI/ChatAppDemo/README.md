# HarmonyOS-WeChat

## 项目简介

这是一个基于HarmonyOS开发的微信客户端应用，采用了模块化设计，包含了微信的核心功能模块。

## 目录结构

```
entry/src/main/
├── ets/                    # 业务逻辑代码目录
│   ├── components/         # 自定义组件
│   ├── entryability/       # 应用入口能力
│   ├── entrybackupability/ # 应用备份能力
│   ├── pages/              # 页面组件
│   └── viewmodel/          # 数据模型
├── resources/              # 资源文件目录
│   ├── base/               # 基础资源
│   └── dark/               # 暗黑模式资源
└── module.json5            # 应用模块配置文件
```

## 目录及文件说明

### ets/ 目录

业务逻辑代码目录，采用TypeScript编写，包含应用的所有功能实现。

#### components/
自定义组件目录，包含应用中复用的UI组件：

| 文件名 | 功能描述 |
|--------|----------|
| ChatList.ets | 聊天列表组件，显示最近聊天记录 |
| Contact.ets | 联系人组件，显示联系人列表 |
| ContactGroup.ets | 联系人分组组件，用于通讯录分组展示 |
| NavTitle.ets | 导航标题组件，用于页面顶部导航 |
| Post.ets | 动态发布组件，用于朋友圈动态展示 |
| Profile.ets | 个人资料组件，显示用户个人信息 |

#### entryability/
应用入口能力目录，管理应用的生命周期：

| 文件名 | 功能描述 |
|--------|----------|
| EntryAbility.ets | 应用主入口能力，处理应用的创建、销毁、前台后台切换等生命周期事件 |

#### entrybackupability/
应用备份能力目录：

| 文件名 | 功能描述 |
|--------|----------|
| EntryBackupAbility.ets | 应用备份恢复能力，处理应用数据的备份和恢复 |

#### pages/
页面组件目录，包含应用的所有页面：

| 文件名 | 功能描述 |
|--------|----------|
| Index.ets | 应用启动页面，显示登录和注册入口 |
| Login.ets | 登录页面，处理用户登录逻辑 |
| Register.ets | 注册页面，处理用户注册逻辑 |
| Main.ets | 应用主页面，包含底部标签栏和四个主要功能模块 |
| Chat.ets | 聊天详情页面，处理单聊或群聊功能 |

#### viewmodel/
数据模型目录，包含应用的数据结构和业务逻辑：

| 文件名 | 功能描述 |
|--------|----------|
| ContactModel.ets | 联系人数据模型，管理联系人信息 |
| Message.ets | 消息数据模型，定义消息的结构和类型 |

### resources/ 目录

资源文件目录，包含应用的所有资源文件。

#### base/
基础资源目录，包含默认主题的资源：

| 子目录/文件名 | 功能描述 |
|--------------|----------|
| element/     | 基础元素资源，包含颜色、尺寸、字符串等 |
| media/       | 媒体资源，包含图片、图标等 |
| profile/     | 配置文件，包含页面路由、备份配置等 |

#### dark/
暗黑模式资源目录，包含暗黑主题下的资源：

| 子目录/文件名 | 功能描述 |
|--------------|----------|
| element/     | 暗黑模式下的元素资源，如颜色定义 |

### module.json5

应用模块配置文件，定义应用的基本信息、页面路由、权限等：

- 应用名称、类型、描述
- 主入口能力配置
- 支持的设备类型
- 页面路由配置
- 权限声明

## 应用功能模块

1. **微信聊天**：显示聊天列表和聊天详情，支持消息发送和接收
2. **通讯录**：管理联系人信息，支持联系人分组
3. **朋友圈动态**：发布和查看朋友圈动态
4. **个人中心**：管理个人资料和设置
5. **用户认证**：支持用户注册和登录

## 技术栈

- HarmonyOS 4.0+
- ArkTS 语言
- ArkUI 框架

## 开发环境

- DevEco Studio
- HarmonyOS SDK

## 运行方式

1. 使用 DevEco Studio 打开项目
2. 连接 HarmonyOS 设备或启动模拟器
3. 编译并运行应用

## 项目状态

该项目处于开发阶段，目前实现了基本的微信客户端功能框架。
